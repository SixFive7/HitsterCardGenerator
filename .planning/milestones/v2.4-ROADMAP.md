## Overview

Add playlist builder as alternative to CSV upload - search Spotify and build a playlist directly in the app.

## Phases

### Phase 17: Spotify Search

**Goal**: Backend search endpoint + search UI component for Spotify track discovery
**Depends on**: Phase 16 (stable codebase with Docker/CI)
**Plans**: 1 plan

Plans:
- [x] 17-01: GET /api/search endpoint with SpotifySearch.svelte component

**Details:**
- Search API endpoint at GET /api/search?q={query} returning up to 20 Spotify tracks
- SpotifySearch.svelte component with 300ms debounced input
- Results display with 64x64 album artwork thumbnails matching app aesthetic

### Phase 18: Playlist Builder

**Goal**: Playlist store + PlaylistBuilder component for track selection and genre assignment
**Depends on**: Phase 17
**Plans**: 1 plan

Plans:
- [x] 18-01: Playlist store with add/remove/genre, SpotifySearch "+" button, PlaylistBuilder component

**Details:**
- PlaylistTrack interface extending SearchResult with genre field
- playlist.svelte.ts store with addTrack, removeTrack, updateTrackGenre, clearPlaylist functions
- "+" button on SpotifySearch results with checkmark for already-added tracks
- PlaylistBuilder component with genre dropdown, remove button, and continue action

### Phase 19: Flow Integration

**Goal**: Landing page dual-path choice, skip matching for built playlists
**Depends on**: Phase 18
**Plans**: 1 plan

Plans:
- [x] 19-01: Landing page dual-path choice with playlist-to-preview direct flow

**Details:**
- Landing page transformed from single "Get Started" to two equal path cards (Upload CSV / Build Playlist)
- Build step with SpotifySearch + PlaylistBuilder side-by-side layout
- Playlist-to-preview conversion skips Spotify matching entirely
- Flow-aware back navigation respects chosen path
