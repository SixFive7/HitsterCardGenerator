<task type="auto">
  <name>Task 1: Create .NET 10 console project with NuGet packages</name>
  <files>HitsterCardGenerator.csproj, Program.cs</files>
  <action>
    1. Create new .NET 10 console project: `dotnet new console -n HitsterCardGenerator -f net10.0`
       Note: If net10.0 is not available, use the latest preview or net9.0
    2. Add NuGet packages:
       - `dotnet add package Spectre.Console` (UI framework)
       - `dotnet add package QuestPDF` (PDF generation - for later phases)
       - `dotnet add package QRCoder` (QR codes - for later phases)
    3. Verify project builds: `dotnet build`
    4. Create basic Program.cs with placeholder that confirms Spectre.Console works:
       ```csharp
       using Spectre.Console;
       AnsiConsole.MarkupLine("[green]Hitster Card Generator initialized![/]");
       ```
  </action>
  <verify>dotnet build succeeds without errors, dotnet run displays green text</verify>
  <done>Project created, all packages installed, builds and runs successfully</done>
</task>

<task type="auto">
  <name>Task 2: Implement FIGlet header and two-panel layout</name>
  <files>Program.cs, UI/AppLayout.cs</files>
  <action>
    1. Create UI folder for layout components
    2. Create AppLayout.cs with static method to render the main UI:
       - Use FigletText for "Hitster Card Generator" title (use a readable font like "Standard" or "Big")
       - Use Layout class to create two-column structure below the header
       - Left panel: "Steps" - will show numbered step list
       - Right panel: "Content" - will show current step UI
       - Use Panel with borders for visual separation
    3. Update Program.cs to call AppLayout and display the shell
    4. Use AnsiConsole.Write() for the layout (not Live display yet - keep it simple)

    **Spectre.Console patterns to use:**
    - `new FigletText("Title").Color(Color.Blue)` for header
    - `new Layout("Root").SplitRows(...)` for vertical split (header + content)
    - `new Layout().SplitColumns(...)` for horizontal split (steps + content)
    - `new Panel(content).Header("Title")` for bordered sections
  </action>
  <verify>dotnet run displays FIGlet header with two-column layout below</verify>
  <done>FIGlet title renders, two-panel layout visible with "Steps" and "Content" sections</done>
</task>

<task type="auto">
  <name>Task 3: Create step progression model and display</name>
  <files>Models/Step.cs, Models/WizardState.cs, UI/StepMenu.cs, Program.cs</files>
  <action>
    1. Create Models folder
    2. Create Step.cs enum or record representing the 8 wizard steps:
       - ImportCsv, ValidateCsv, SpotifyAuth, SpotifySearch, GenerateQR, ColorChoice, DesignCards, ExportPdf
    3. Create WizardState.cs class to track:
       - CurrentStep (which step is active)
       - CompletedSteps (list of finished steps)
       - Method to advance to next step
    4. Create StepMenu.cs to render the step list:
       - Show all 8 steps numbered 1-8
       - Current step highlighted (e.g., yellow with arrow)
       - Completed steps marked with checkmark (green)
       - Pending steps dimmed
       - Use Spectre.Console Markup for colors: [green]✓[/], [yellow]→[/], [grey]○[/]
    5. Update Program.cs to:
       - Create WizardState
       - Render layout with StepMenu in left panel
       - Show placeholder text in right panel
    6. For now, just display static state (step 1 active, none complete)
  </action>
  <verify>dotnet run shows step menu with step 1 highlighted, steps 2-8 pending</verify>
  <done>Step model created, step menu renders with proper highlighting, state tracking works</done>
</task>
